It has since been removed from the working tree (`.env.galerina-signing` is now git-ignored and untracked) and the active signing identity was rotated to `ab46f4c7e2797b9b`.

Because the key material exists in the immutable git history, it must be treated as **permanently compromised**, regardless of working-tree state.

## Remediation decision: revocation record, NOT history rewrite

A formal revocation is published **in preference to** a `git filter-repo` / BFG history scrub. Rationale: rewriting history rebases every subsequent commit SHA, which **breaks the verifiable trust chain** any auditor relies on in a cryptographic zero-trust project — a worse outcome than the documented exposure. The exposed key lineage is instead permanently distrusted via this advisory. (If a history scrub is ever legally mandated, it must be accompanied by a re-attestation of all post-`cb5036d` artifacts.)
