## The critical path — U1, the WAT-emitter frontier

Everything below marked **[U1]** is unblocked by advancing the WAT emitter. It is the single highest-leverage
investment: it collapses `.ts` removal, `node_modules` removal (typescript retires with `.ts`), DSS.wasm
compilation, and the P9 self-hosting frontier into one program.

## Phase 1 — buildable NOW (no unlock; keep-green increments)

1. **`Int.bitAnd`/`Int.bitOr` → `i32.and`/`i32.or`** in the WAT emitter *(task #62)* **[U1]** — takes DSS.wasm 6/12 → ~12/12 R0 and unblocks every bitwise `.fungi`. Core-compiler change; verify against the 4,505 suite + re-run the DSS probe.
2. **RD-0361 twin queue** *(task #53)* — drive the 18 shadow twins → differential on the proven template (Memory ×5, I/O ×3, rest of TLSTP ×3, Packages ×1, +others). Pure keep-green tests; advances the thesis surfaces.
3. **Benchmark harness standardisation** *(task #63)* — warmup + fixed-time + median-of-N + a fixed-clock rig; makes the "more stable than Rust" claim honest (equal windows). R&D-endorsed.
4. **Security-infra (non-Z3 legs)** — the fuzz differential-oracle (host `.mjs`, leverages the twin differential infra) + the tabletop runbook.
5. **RD-0363/0364 P/I wiring + a first RD-0365 surface** — extend the executing decision surfaces (#105).
6. **Package Standard** — the `.graph/` §2 amendment (doc) + R1–R2 rungs on the pure-logic packages.
7. **ZTF-KB path-leak redaction** — mechanical 346-leak / 101-file redaction (agent edits; owner commits + CI).

## Phase 2 — the P9 self-hosting frontier [U1]

8. **WAT-emitter completion** — lower the remaining constructs stage by stage; drive `parser` → `type-checker` → `effect-checker` → `governance-verifier` to Stage-A == Stage-B **byte parity** (tokenize already proven, #143).
9. **DSS.wasm** — after phase-1 #1: multi-module merge/name-mangle → a real `dss-supervisor.wasm`; then the host-side WASM **Component-Model isolation wiring** (byte-mover ring-buffer, zero-copy). No new R&D. *(Isolation guarantee stays aspirational until the wiring + honest semantics land.)*
10. **The 49-package `.fungi` migration** — one at a time, keep-green (author `.fungi` → lower → tests green → delete the `.ts`). Retires `typescript`/`@types/node` with the `.ts`.
11. **Value-unit types (RD-0349)** — I4–I6 (compiler impl); I1 pending a pinned ISO-4217 data source **[U4]**.

## Phase 3 — owner / hardware gated

12. **R4 authority flip (#143)** **[U3]** — the switch that deletes a differential twin's `.ts` decider and makes the `.fungi`/WASM authoritative. Owner-gated capability to build.
13. **Signing-key custody L2–L4** **[U6]** — key bytes → vault → TPM → HW. Owner + hardware; the agent never handles private keys. Agent-side ceiling = the `env.spore` format + runbook (done/near-done).
14. **Z3 assurance model** **[U5]** — needs a ZT-21 native-floor ruling (Z3 is a native/`node_modules` dep) or a pure-`.mjs` bounded checker instead.

## Phase 4 — post-v1

15. **Crypto-floor self-hosting** — unfloors the 36 floored packages (`@noble/*`, `argon2`, `bcryptjs`, `snarkjs` → `.fungi`/vetted native seam).
16. **Self-hosting bootstrap fixpoint** (Stages 3–6) — the terminal milestone that lets the **compiler delete its own `.ts`** (it is the bootstrap compiler; it can only self-delete once a self-hosted `.fungi` compiler passes a fixpoint).
17. **Real DSS.wasm Wasmtime TCB (#102–106)** — deterministic isolation / in-sandbox decryption; the true kernel-bypass property.
18. **Photonic / Ternary + AI Inference Tower + Passive bridges** — the hardware-tier frontier (currently 3–40%).
19. **TritMesh / `.hypha` / MeshQL** — the next project, built ON Galerina once it's in production.

## Product direction — Galerina:HotPath (GHP)  [design-stage · v1-vs-v2 owner decision]

Beyond the self-hosting / frontier work above, an R&D handover proposes **Galerina:HotPath (GHP)** — a deterministic,
allocation-free, low-latency execution component *inside* the runtime (RD-0375; buildable pieces **RD-0376** Hallmark
Seal two-mode split + **RD-0382** execution-order arena). It is a **runtime capability, not DB-only**, and a **v1
candidate if a use-case justifies it, else v2**. Security spine (fail-closed): HotPath is **K3-admitted** (a
specialisation, never a bypass), the `FASTPATH` is **planner-proven, not user-forceable**, and a seal keeps
`overflow: trap` + bounds + the RD-0358 residency floor. Governance rules: execution lanes (HOT/STANDARD/OFFLOAD/
DENIED) **must not alias** the K3 trust verdict; a seal is a *specialisation*, never a governance exemption. Honest
tier: **sub-microsecond bounded**, not nanosecond latency (1 ns ≈ 4 cycles). Strengthens value-unit types (RD-0349)
+ hallmark (RD-0353). Tracked as **task #69**. *The TritMesh product family + the ns-DB aspiration are a **separate
next project** (design-only) — not this.*
