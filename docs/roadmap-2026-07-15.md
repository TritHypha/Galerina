## The critical path — RD-0389 record ABI (✅ ARG dir DONE), then DSS merge + U1

The prior critical path (#62, #64, #68) is **done**. What's left needs engineering, not research.

1. **✅ Record-marshalling ABI — RD-0389 (ARG direction) DONE (2026-07-15, commit `e40e78d7`).** Closed the last
   non-P9 twin shadow: `secret-gate` (`Array<SecretPresence>` arg) now EXECUTES and its verdict ≡ the shipped
   `createSecretGate().admit()` over a 14-scenario grid. **Grounding correction to the plan above:** a WAT probe
   showed the twin BODY already lowers completely (`Array<record>` access, `match`-Option, and `s.field` →
   `i32.load(ptr+off)` all resolve; assembles VALID) — so the predicted 4-step *emitter* change was NOT needed. And
   a *module-exported* `__mk_<R>` would fight the B2/B2b per-invocation arena (records built cross-call get
   reset/zeroed at the consuming flow's entry — a silent fail-open). The correct path was **host-side marshalling,
   no core-emitter change**: two test/differential-only `HostRuntime` methods in `wasm-runtime.ts` — `internArray`
   (the array of record ptrs) and `allocRecord` (stage a record in the module's own exported linear memory from a
   `WAT_HEAP_BASE` bump pointer, fail-closed) — the ARG-direction mirror of the existing `readArray`/`readRecordField`
   READ bridge (the same move #64/#68 made for strings). Kept the suite green: **92/92 · 7,124.** secret-gate 26/2 →
   **27/1**. The `__rd_`/readback direction the P9 parser needs already exists (`readRecordField`, used by the
   tokenize twin) → that's the U1 frontier increment (item 3), not a new ABI.
2. **DSS.wasm multi-module merge** *(the 12th module + the real `.wasm` artifact)* — a link stage over the import
   graph (shared intern table + record-layout map) vs multi-instance host-brokered composition (Component-Model,
   matches #102-106 isolation). Both reuse the 11 green modules.
3. **U1 — WAT-emitter frontier (P9):** the remaining constructs, driving `parser` → `type-checker` →
   `effect-checker` → `governance-verifier` to Stage-A == Stage-B **byte parity** (tokenize proven, #143). This is
   also the gate that retires `.ts` + `typescript`/`@types/node`.

## Product direction — Galerina:HotPath (GHP)  [design-stage · v1-vs-v2 owner decision]

An R&D handover proposes **Galerina:HotPath (GHP)** — a deterministic, allocation-free, low-latency execution
component *inside* the runtime (RD-0375; buildable pieces **RD-0376** Hallmark Seal two-mode split + **RD-0382**
execution-order arena, now grounded by the RD-0370 memory-layout research: one 64-B-aligned arena in execution
order, prefetcher-friendly, ~0 transition misses). A **v1 candidate if a use-case justifies it, else v2**. Security
spine (fail-closed): HotPath is **K3-admitted** (a specialisation, never a bypass); the `FASTPATH` is
**planner-proven, not user-forceable**; a seal keeps `overflow: trap` + bounds + the RD-0358 residency floor;
execution lanes (HOT/STANDARD/OFFLOAD/DENIED) **must not alias** the K3 trust verdict. Honest tier:
**sub-microsecond bounded**, not nanosecond. Tracked as **task #69**.
