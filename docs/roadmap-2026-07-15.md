## The critical path — now two ABI/merge problems + U1

The prior critical path (#62, #64, #68) is **done**. What's left needs engineering decisions, not research:

1. **Record-marshalling ABI** *(unblocks the last 2 twin shadows AND P9 parser byte-parity)* — the `secret-gate`
   twin (Array<record> args) and the P9 parser (nested Expr/Stmt AST readback) both need a JS↔linear-memory
   record path. Cleanest: emit a per-record constructor export `__mk_<Record>(...) -> ptr` (pure, no host state,
   nested-safe; the emitter already knows every layout). One design closes both. *(R&D ask #1.)*
2. **DSS.wasm multi-module merge** *(the 12th module + the real `.wasm` artifact)* — a link stage over the import
   graph (shared intern table + record-layout map) vs multi-instance host-brokered composition (Component-Model,
   matches #102-106 isolation). Both reuse the 11 green modules. *(R&D ask #2.)*
3. **U1 — WAT-emitter frontier (P9):** the remaining constructs, driving `parser` → `type-checker` →
   `effect-checker` → `governance-verifier` to Stage-A == Stage-B **byte parity** (tokenize proven, #143). This is
   also the gate that retires `.ts` + `typescript`/`@types/node`.

## Product direction — Galerina:HotPath (GHP)  [design-stage · v1-vs-v2 owner decision]

An R&D handover proposes **Galerina:HotPath (GHP)** — a deterministic, allocation-free, low-latency execution
component *inside* the runtime (RD-0375; buildable pieces **RD-0376** Hallmark Seal two-mode split + **RD-0382**
execution-order arena, now grounded by the RD-0370 memory-layout research: one 64-B-aligned arena in execution
order, prefetcher-friendly, ~0 transition misses). A **v1 candidate if a use-case justifies it, else v2**. Security
spine (fail-closed): HotPath is **K3-admitted** (a specialisation, never a bypass); the `FASTPATH` is
**planner-proven, not user-forceable**; a seal keeps `overflow: trap` + bounds + the RD-0358 residency floor;
execution lanes (HOT/STANDARD/OFFLOAD/DENIED) **must not alias** the K3 trust verdict. Honest tier:
**sub-microsecond bounded**, not nanosecond. Tracked as **task #69**.
