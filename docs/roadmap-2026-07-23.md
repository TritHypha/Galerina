**Landed since the early-2026-07-23 round** (each verified at commit time):

- **Token-kind semantics arc — LIVE-COMPLETE.** The root defect was a dispatch wildcard: unmapped value
  types collapsed to one comparison bucket, so `==` between them fail-*open* (`{x:1} == {x:2}` → true) and
  the self-hosted twins mis-read `Token.kind == "…"`. The composite-equality fail-open is closed (records
  and lists now compare structurally, `195804fd`); the live twin sites are migrated to value-based /
  positional reads that compile identically on both backends (underscore lexer `7d603948`, policy branch
  `d622b464`, guard-name + conforms_to `72e85535`) — no broken `Token.kind == "string"` site remains. The
  deeper **a1** (full enum-value evaluation) is deferred as a *coordinated* interp + WASM-emitter change:
  fixing only the interpreter would break the standing R3 interp≡WASM byte-parity gate, because the wildcard
  is identical in both backends.
- **Crypto-agility (RD-0099) — part 1 landed.** `governance/crypto-suites.json` mirrors the authoritative
  signature-suite register across all three domain-separated families (governance-signature, audit-
  attestation, bridge-manifest); the reader `governance/crypto-suites.mjs` is fail-closed — only an
  `active-for-signing` suite may sign, a `verify-only-retired` suite keeps its verifier (the iron rule), and
  an unknown suite or a missing register denies (`eec56499`, `1c437ca`; 18/18). It carries a measured
  security finding for the part-3 gate: `signProofGraphHybrid` silently downgrades to Ed25519 v1 on a
  non-hybrid key — asymmetric with the verifier's no-PQ-downgrade rule.
- **Compiler-stage evidence is now a gated baseline** (`4df91ec1`): the 7 stage hashes are re-derived and
  drift-checked in phase-close, replacing the stale point-in-time snapshot.
- **I-3 functional oracle — three tranches** over the real self-hosted pipeline (accept-good / reject-bad,
  every code measured, strictly pre-authority): type-correctness (FUNGI-TYPE), parse-correctness
  (FUNGI-PARSE-001..004, `0f3a7761`), and governance-correctness (FUNGI-GOV-002 + FUNGI-VAL-001 via
  parse→verifyGovernance / checkBodyGovernance, `e1a9112c`). 27/27.

**Why this order:** the token-kind arc that headed the previous roadmap is done, so the critical path is now
the self-hosting track. Finding (d) leads because it is a measured parser-flip *precondition* — a flipped
self-hosted parser that drops `classification` would silently skip the safety-critical governance checks.

sequenced before the parser flip, not before everything. Crypto-agility 2–3 runs in parallel as the
owner-commissioned track and reuses the part-1 register + the drift-gate skeleton.
