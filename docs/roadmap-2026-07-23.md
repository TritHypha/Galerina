**Why this order:** the measured ground truth (2026-07-23) is that enum comparison is broken in *every*
form — `Color.Red == Color.Green` evaluates `true`, `== "Red"` is always false, bare members are
always true, and the type-checker is silent on all of it. Migrating parser sites onto qualified names
before (a1) would land on the same broken semantics. Acceptance for rung 1 is the probe table flipping to
`true / false / error / error / error`.
