## In flight — the flagship (owner-commissioned, 0119 item 2)

**Three-valued parameter admission** — `name: Type where <k3-expr>`, a Verdict-ALLOW-only entry gate. The
design is now fully grounded at source (2026-07-24); COMMIT 1 is a single indivisible fail-closed unit:

1. `parseParam` parses `where <k3-expr>` (params today have **no** `where` — only `readonly`/`tainted` +
   `name: Type` + optional `source_from Origin`).
2. The predicate attaches as a **sibling flow-clause** keyed by param name (carry design **(B′)**), NOT as a
   `paramDecl` child — `paramDecl` has 36 consumers across 14+ files, and a synthesized `contractDecl` would
   flip `NodeFlags.HasContract`. (B′) leaves both untouched, making the regression *unrepresentable*.
3. `checkParameterAdmission` — a **sibling** of `checkInputPreconditions` (`interpreter.ts:1509`), deliberately
   stricter: Verdict-ALLOW-only (`__tag==='verdict' && value===+1`), **no int-truthiness** (A7 accepts
   `int !== 0`; admission must not).
4. **Fast-tier exclusion** — admission-bearing flows must decline all four fast tiers (bytecode VM `:3653`,
   `executeFlowSync` `:3532`, ExecutionGraph fast-path `:1337`, pure-flow cache `:3844`), exactly as
   invariant-bearing flows already do via `flowHasEnforcedInvariants` (`:3371`). **Without this the gate is
   inert** — production silently walks around it. (Independently confirmed by R&D.)
5. Type-check Bool|Verdict auto-lift · `FUNGI-ADMIT-001/002/003` · a real-source `where all{}` → UNKNOWN →
   REFUSE test.

**Owner decision pending:** COMMIT 1 spans two large files + a new test + a full parity pass and does not fit a
15-minute tick; the tick rules forbid splitting a commit or leaving a dirty tree. Needs one of: suspend
tick-bounding for this item · authorize a parse+**REFUSE** rung 1 (fail-closed, complete alone) then a rung 2
that swaps in the live gate · or reprioritise. Until then the flagship is parked, not abandoned.
