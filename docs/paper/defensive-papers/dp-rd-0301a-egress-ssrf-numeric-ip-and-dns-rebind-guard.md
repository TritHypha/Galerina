# Defensive Publication — Deny-by-default SSRF egress guard: numeric/IPv6-embedded-IPv4 obfuscation normalization + connect-time per-resolved-address DNS-rebind recheck

**Disclosure ID:** DP-RD-0301a **(PROPOSED — main session to confirm/assign the final number and land into `docs/paper/defensive-papers/` + update the paper index)** · **Date:** 2026-07-10 · **Type:** Prior-art disclosure (defensive) · **Not** a patent claim.
**Provenance:** RD-0301 (flagged this construction as detailed/novel enough for its own disclosure, previously unpublished) · construction `packages-galerina/galerina-core-network/src/egress-guard.ts` · unit tests `packages-galerina/galerina-core-network/tests/egress-guard.test.mjs` · compiler-wiring regression `packages-galerina/galerina-core-compiler/tests/stdlib/ssrf-egress-guard.test.mjs` (self-audit finding 61-9) · re-runnable proof `packages-galerina/galerina-core-network/scripts/prove-egress-guard.mjs`.

> **Purpose.** Defensive publication; **novelty disclaimed** (§3). The recorded construction is a deny-by-default outbound (egress) SSRF guard with three load-bearing properties: (1) every IPv4/IPv6 numeric-literal encoding of a given address — decimal, hexadecimal, octal, inet_aton-style partial-dotted, and the IPv4-mapped/compatible/translated/NAT64/6to4 IPv6 embeddings — is normalized to the same canonical four-octet form **before** range classification, so a single range table catches every disguise of the same address, including the compact hex-hextet form the WHATWG URL parser itself produces when canonicalizing a bracketed IPv4-mapped literal; (2) a hostname verdict is never final — it carries a `requiresDnsRecheck` flag, and a companion function re-classifies **every** address the caller's own resolver returns, denying if any one is non-public, closing DNS-rebinding/mixed-resolution TOCTOU at the point of resolution; (3) the cloud-metadata range is gated by its own flag independent of the general non-public-host permission, embedded URL credentials are denied by default, and transport/port checks run strictly after host-category classification so a non-public-host denial keeps its own finding code.

---

## 1. Technical field

Server-Side Request Forgery (SSRF) defense in language/interpreter runtimes and API gateways that dial attacker-influenceable outbound URLs (webhooks, image-fetch, redirect targets, `http.*` stdlib calls); IP-literal parsing and normalization against numeric-encoding bypasses; DNS-rebinding defense at connect time; egress transport-security (TLS/port) policy enforcement.

## 2. Background & problem

SSRF (CWE-918; OWASP API Security Top 10, API7:2023) arises when a server dials a URL under attacker influence and that URL is pointed at the server's own private network or a cloud metadata endpoint (`169.254.169.254` and equivalents) to exfiltrate credentials. Naive defenses — a prefix/regex check against known-bad string forms — are a well-documented, repeatedly-bypassed pattern, for two independent reasons:

1. **Numeric-encoding bypass.** IPv4 parsers descended from BSD `inet_aton` accept decimal (`2130706433`), hexadecimal (`0x7f000001`), octal (`0177.0.0.1`), and short/partial dotted forms (`127.1`) as `127.0.0.1`; a string check written against the dotted form misses all of these. IPv6 compounds this: an IPv4 address can be embedded via IPv4-mapped/compatible/translated (`::ffff:W:Z`), NAT64 (`64:ff9b::/96`), or 6to4 (`2002:V4::/16`) forms.
2. **Parser-canonicalization bypass.** The WHATWG `URL` global itself rewrites a bracketed dotted IPv4-mapped literal into the compact hex-hextet form: `new URL("https://[::ffff:169.254.169.254]/").hostname === "[::ffff:a9fe:a9fe]"`. A check that only recognizes the dotted metadata address never fires on what the runtime actually receives after parsing — a check can be *individually correct* against the input it was tested with and still miss the input the URL parser hands it in production. This exact case is marked `AUDIT REGRESSION (critical SSRF)` in the test suite.
3. **DNS rebinding / TOCTOU — the classic time-of-check/time-of-use case.** A hostname that resolves to a public address at check time can resolve to a private/metadata address at connect time (attacker-controlled authoritative DNS, short TTL), or resolve to a *mix* of public and private addresses. A guard that only classifies the hostname string proves nothing about the address actually dialled.

The concrete motivating regression (self-audit finding 61-9): a live interpreter `http.*` path previously used an inline private-range regex that **missed** IPv4-in-IPv6 metadata, `*.internal`/`*.corp` TLDs, CGNAT `100.64/10`, decimal/hex numeric IPs, and embedded-credential URLs — a reproduced, not hypothetical, defect this construction replaced.

The range table itself is standard; not claimed. The cloud-metadata address `169.254.169.254` is a cross-provider convention (AWS/GCP/Azure/etc.), not an IETF allocation — cited as such, not as an RFC.

- **The WHATWG URL Living Standard's IPv6 canonicalization behaviour** — standard, specified parser behaviour; cited as the source of the specific bypass form in §2, not claimed.
- **DNS-rebinding as a named attack class** — well established (browser anti-rebinding pinning, "time-of-check-to-time-of-use" literature generally). Not claimed.
- **Sibling disclosures:** DP-RD-0269 (fail-open taxonomy — a string/regex-only SSRF check, and a DNS-rebind check that exists but is never actually invoked, are both structural instances of the classes that taxonomy names; the 61-9 regression this note cites is itself a reproduced defect in that spirit); DP-RD-0216 (a deny-only pre-filter is forgeable/bypassable unless it *is* the deny path — informs why every classification path here terminates in an explicit allow/deny, never an implicit pass); DP-RD-0295a (coverage-scorecard framing: the numeric/IPv6-embedding bypass class is closed **unrepresentable-by-construction** once normalization precedes classification; DNS rebinding remains **detect/recheck-only**, per the honest limitations below, not eliminated).

**Novel only in combination:** normalizing *all* IPv4 numeric encodings and *all four* documented IPv6 embedded-v4 families to one canonical-octet classification path shared with the plain-dotted form; a `requiresDnsRecheck` flag structurally threaded from the hostname classifier through to a dedicated resolved-address recheck function so a pre-resolution verdict cannot silently stand in for a post-resolution one; a metadata-specific gating flag independent of the general non-public-host flag; and a fixed check order (host-category → transport → port) that preserves the SSRF finding's identity through later checks — combined in one guard, with a reproduced regression (61-9) and a reproduced parser-canonicalization bypass as the evidence base rather than a hypothetical threat model.

## 4. Summary of the disclosed subject matter

A pure, side-effect-free TypeScript module (`egress-guard.ts`) exporting: `classifyHost(rawHost)`, which normalizes an IPv4/IPv6/hostname string (via `parseOctet`, `ipv4ToOctets`, `expandIpv6`, `classifyIpv4`, `classifyIpv6`, `classifyHostname`) into a `HostClassification` carrying an eleven-member `HostCategory` (`loopback | private | linkLocal | uniqueLocal | cgnat | metadata | multicast | unspecified | reserved | broadcast | public | invalid`) and a `requiresDnsRecheck` boolean; `guardOutboundHost(rawHost, policy)`, a deny-by-default decision over a `DENY_CATEGORIES` set (every category except `public`), with independent opt-in flags for non-public hosts, the metadata endpoint specifically, an exact-host allow-list, and a narrow local-development loopback exception; `guardOutboundUrl(url, policy)`, which additionally enforces scheme allow-listing, denies embedded userinfo credentials by default, and optionally enforces required-TLS and a port allow-list *after* host-category classification; `guardResolvedAddresses(host, resolvedIps, policy)`, the connect-time DNS-rebinding defence, which re-classifies every caller-supplied resolved address and denies on the first non-public one; and `validateWebhookTarget(url, policy)`, a strict composition (https-only, no metadata, no non-public, no credentials) for the highest-risk caller-supplied-URL case, that still honours an explicit allow-list.

## 5. Detailed description

### 5.1 Deny-by-default classification with metadata gated independently
`DENY_CATEGORIES` denies every category except `public`; only a provably-public destination (or an exact allow-listed host) is dialled by default. The cloud-metadata range is checked **first**, inside the wider `169.254.0.0/16` link-local block (`classifyIpv4`: `a===169 && b===254 && c===169 && d===254` before the general `169.254/16` rule), and is gated by its own `allowMetadataEndpoint` flag, distinct from `allowNonPublicHosts` — verified by test: *"the metadata endpoint is denied even with `allowNonPublicHosts` (needs its own explicit flag)."* This means opening private-range egress for a legitimate internal integration does not implicitly open the single highest-value SSRF target.

### 5.2 Numeric-IPv4 obfuscation defeat
`parseOctet` accepts hex (`0x[0-9a-f]+`), octal (`0[0-7]+`), and decimal forms per octet; `ipv4ToOctets` additionally implements `inet_aton`'s partial-dotted absorption (the *last* part absorbs the remaining low-order bytes across 1–4 dotted parts) and the single-integer 32-bit form. All of `2130706433`, `0x7f000001`, `0177.0.0.1`, `127.1`, and `127.0.1` classify identically to `127.0.0.1` — loopback — because they are converted to the same four-octet array *before* `classifyIpv4` runs a single range table over it. There is exactly one range table, not one per encoding.

### 5.3 IPv6-embedded-IPv4 defeat, including the URL-parser-canonicalization bypass
`expandIpv6` handles `"::"` compression, a zone id (`%eth0`, stripped), and an embedded dotted-v4 tail, converting the latter to two hex groups before expansion. `classifyIpv6` then explicitly reclassifies via `classifyIpv4` for four embedded-v4 families: IPv4-mapped/compatible/translated (`::ffff:W:Z` and variants), NAT64 (`64:ff9b::/96`), and 6to4 (`2002:V4::/16`), plus a hard-coded AWS IPv6 metadata literal (`fd00:ec2::254`). The module's own comment states the specific, previously-real regression this closes: *"This closes the hex-hextet bypass that the WHATWG URL parser produces (`new URL("https://[::ffff:169.254.169.254]/").hostname === "[::ffff:a9fe:a9fe]"`)"* — reproduced in the test suite as `AUDIT REGRESSION (critical SSRF)`, asserting `::ffff:a9fe:a9fe → metadata` and its bracketed/fully-expanded/NAT64/6to4 equivalents. A malformed IPv6 literal returns `"invalid"`, not `"public"` — the parser fails **closed** on unparseable input rather than defaulting it to open.

### 5.4 DNS-rebinding / TOCTOU defence via structural recheck
Every hostname classification carries `requiresDnsRecheck: true` (bare hostnames, `.local`/`.internal`/`.intranet`/`.corp`/`.home.arpa`, and tentatively-public FQDNs all set it; IP literals and allow-listed hosts do not). `guardResolvedAddresses(host, resolvedIps, policy)` is the connect-time half: it re-classifies **every** address the caller's resolver returned and denies on the first non-public one — *"mixed resolution is an attack"* — and fails closed on zero resolved addresses. The module's own doc-comment states the caller's obligation precisely: *"The caller passes the addresses its resolver returned and MUST connect ONLY to a verified address (pin it; do not re-resolve)."*

### 5.5 Credential/transport/port posture and check ordering
`guardOutboundUrl` denies embedded userinfo credentials by default (`allowUrlCredentials`, default `false` — *"parser-confusion SSRF vector"*), and, for an otherwise-allowed host, enforces `requireTls` (denying an otherwise-permitted plaintext scheme) and `allowedPorts` (scheme-default-aware: `https`⇒443, `http`⇒80) — but only **after** host-category classification. Test-verified ordering property: an internal host requested over plaintext under `requireTls` still reports `Galerina_NETWORK_SSRF_NONPUBLIC_DENIED`, not a TLS finding — a later, narrower check never masks an earlier SSRF finding's identity. `allowLoopback` opens *only* loopback (`127.0.0.0/8`/`::1`/`localhost`) for local development, explicitly excluding private LAN/metadata/link-local; an exact-host `allowedHosts` entry is treated as an operator trust decision and bypasses TLS/port checks for that host only (an internal egress proxy), in every environment including production.

## 6. Honest limitations & scope

- **The classifier's fail-closed property is bounded by the currently-enumerated range table, not a closed-form proof over all of IPv4/IPv6 space.** Both `classifyIpv4` and `classifyIpv6` end in an unconditional `return cls(host, "public", ...)` fallthrough for anything not matched by an earlier rule — the necessary shape for any finite special-purpose-range enumeration (public is the *complement* of the enumerated blocks), but it means a future IANA special-purpose allocation not yet added to this table would classify, and therefore dial, as public until the table is updated. This is a maintenance-currency residual shared by every deny-list-of-special-ranges classifier, not unique to this one; unlike a string check, unparseable input does fail closed (`"invalid"`, denied).
- **A DNS-rebind recheck closes the window only when the validated address is pinned into the actual socket connect.** A recheck-then-reconnect design that re-resolves the hostname at connection time retains an inherent time-of-check-to-time-of-use residual — a property of the DNS-rebind-defence *class*, not of any one implementation. Full closure requires pinning the validated IP into the socket (a resolver/dispatcher override), so that the address classified and the address dialled are provably the same. *(Implementation-specific detail on this codebase's current call-site wiring is withheld from this public disclosure and tracked internally.)*
- **A provably-public destination is exactly what this construction allows — including a public host an attacker fully controls.** Deny-by-default-to-public stops the deployment's *own* private network and metadata endpoint from being reached; it does not and cannot distinguish a legitimate public API from an attacker's own public VPS. Exfiltration-to-attacker-infrastructure over an allowed public egress is out of scope by design, not an oversight.
- **DNS resolver integrity is out of scope.** The rebind defence re-checks *whatever the caller's resolver returns*; it does not defend the resolver itself against poisoning/compromise (DNSSEC and similar are a distinct, unaddressed layer).
- **A pre-filter remains a pure decision, not a network interposition (DP-RD-0216's caveat applies structurally):** nothing in this module *is* an egress proxy — its safety depends on every egress call site actually invoking it. A disclosure of the construction is not a proof that every call site in a given deployment invokes it; that is a caller-side, deployment-specific obligation.
- **Not claimed:** that this construction makes the deployment "unhackable" against SSRF, or that it is a substitute for network-layer egress controls (firewall/NAT egress policy) as defence in depth.

## 7. Illustrative disclosure claims (prior-art disclosures, not patent claims)

1. **A method** of classifying a network egress target wherein a hostname or IP-literal string is normalized — parsing decimal, hexadecimal, and octal numeric encodings and partial-dotted forms of an IPv4 address, and IPv4-mapped, IPv4-compatible, IPv4-translated, NAT64, and 6to4 embeddings of an IPv4 address within an IPv6 literal — to one canonical four-octet representation prior to range classification, such that every encoding of a given address receives an identical category verdict from a single range table.
2. **A method** as in claim 1 wherein the classification enumerates a cloud-metadata category distinct from, and gated independently of, a general non-public-host category, such that a policy permitting non-public-host egress does not implicitly permit metadata-endpoint egress.
3. **A method** as in claims 1–2 wherein a hostname classification carries a recheck flag, and a second function re-classifies every address returned by a resolution performed by the caller, denying if any resolved address falls outside the public category, such that a classification computed before resolution is never treated as authoritative for the connection subsequently made.
4. **A method** as in claims 1–3 wherein a URL carrying embedded userinfo credentials is denied absent an explicit override, and a required-transport-security check and a port allow-list check are evaluated only after host-category determination on an otherwise-permitted request, such that a host-category denial is reported under its own finding identity rather than a later-evaluated transport or port finding.
5. **A method** as in claims 1–4 wherein an HTTP client applying the foregoing suppresses automatic redirect-following and re-applies claims 1–4 in full to each redirect's resolved target before the client is permitted to follow it.
